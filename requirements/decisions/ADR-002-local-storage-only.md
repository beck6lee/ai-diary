# ADR-002 数据仅存 localStorage，不建后端

## 背景

需要决定用户数据的存储方案。选项包括：纯本地、有后端的云存储、或混合方案。

## 备选方案

### 方案 A：localStorage（已选择）

**优点：**
- 零后端成本（无服务器、无数据库）
- 零隐私风险（数据不离开用户设备）
- 开发最简单，无需处理账号/鉴权/同步
- API Key 也由用户自持，无泄漏风险

**缺点：**
- 换设备或清浏览器缓存后数据丢失
- 多设备无法同步
- 浏览器存储容量有限（一般 5-10MB，日记文本足够用）

### 方案 B：Supabase / Firebase 等 BaaS（已放弃）

**优点：**
- 多设备同步
- 数据不会因浏览器清除而丢失

**缺点：**
- 需要账号体系，用户注册门槛增加
- 有月度费用（免费额度用完后）
- 需要处理数据加密（日记属隐私内容）
- 开发工作量增加 2-3 倍
- MVP 阶段不必要的复杂度

### 方案 C：自建后端（已放弃）

成本最高，MVP 完全不考虑。

## 最终决策及理由

**选择方案 A（localStorage）**。

MVP 是个人工具，目标是验证"AI 整理是否好用"，而非验证"多设备同步是否必要"。个人工具用本地存储是合理的取舍。数据丢失风险通过在设置页明示（"数据保存在本地设备"）来管理用户预期。

DeepSeek API Key 的存储同理：明文存 localStorage，风险自担，在设置页注明。

## 影响与后续

- 用户数据无法跨设备访问
- 产品无法统计用户行为（无服务端）
- Phase 3 如需云同步，需要重新设计数据层

## 回顾时机

- 用户频繁反映"换手机后数据丢失"
- 产品进入 Phase 3（规模化阶段）
