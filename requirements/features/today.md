# 今日记录页

## 功能概述

用户进入 App 后默认看到的页面。支持全天多次输入，每次提交后 AI 自动将当天所有记录合并整理成一篇结构化日记，并自动保存。输入栏固定在屏幕底部，方便拇指操作。

---

## 用户故事

- 作为用户，我希望随时随地输入几句话，不需要考虑格式，让 AI 帮我整理
- 作为用户，我希望一天内可以多次补充记录，所有内容汇总到同一篇日记
- 作为用户，我希望每条记录都有时间标记，方便事后回顾
- 作为用户，我希望对 AI 整理结果不满意时，可以手动修改或重新整理
- 作为用户，我希望能看到自己的原始输入内容，确认 AI 没有遗漏

---

## 当前需求

### 页面状态

页面有三种状态，相互切换：

| 状态 | 触发条件 | 展示内容 |
|------|----------|----------|
| `idle` | 默认 / 整理完成 / 保存后 | 日记卡片（Markdown）+ 原始记录折叠区 + 底部输入栏 |
| `formatting` | 点击发送按钮 | 日记卡片变暗→流式替换 + 底部"AI 整理中" |
| `editing-result` | 点击"编辑"按钮 | 可编辑 textarea + 底部"重新整理/保存修改" |

### 头部

- 显示今天日期（格式：M月D日，如"3月27日"）
- `idle` 且有内容时：右侧显示"编辑"按钮
- `editing-result` 时：右侧显示"取消"按钮

### 日记卡片

- `idle` / `formatting` 状态下显示已整理内容（Markdown 渲染）
- `formatting` 时卡片变暗（opacity 0.3），收到第一个流式字符后恢复正常并开始逐字替换
- 流式输出时末尾显示 `▌` 光标
- 无内容时展示空状态引导（图标 + 文案）

### 原始记录（折叠区）

- 仅在 `idle` 状态且有原始内容时展示
- 标题栏："原始记录（N 条）"，点击展开/收起
- 每条原始记录独立展示，保留 `【HH:MM】` 时间前缀
- 条目之间以 `---` 分隔符区分

### 底部输入栏（`idle` 状态）

- textarea（默认 2 行，圆角胶囊形）+ 圆形发送按钮
- 未输入内容时发送按钮禁用
- 占位提示：
  - 当天无记录：「随便写，口语化就行…」
  - 已有记录：「再记一条…」

### 时间戳

- 每次提交时自动在内容前加 `【HH:MM】`（24小时制）
- 多条记录之间用 `\n\n---\n\n` 拼接，共同存入 `raw`

### 编辑模式（`editing-result`）

- 将 `formattedContent` 加载到 textarea 供用户修改
- 「重新整理」：用已有 `rawAccumulated` 重新请求 AI（不追加新内容）
- 「保存修改」：将编辑后的内容保存，跳回 `idle`
- 「取消」：放弃修改，跳回 `idle`

### 错误处理

| 场景 | 处理方式 |
|------|----------|
| API Key 未填写 | 顶部提示「请先在设置页填写 DeepSeek API Key」|
| 请求失败 / 网络错误 | 顶部提示错误信息，rawAccumulated 回滚，恢复上次保存内容 |
| 整理中断 | 同上，不保留不完整内容 |

---

## 数据结构

```js
// localStorage key: "diary_YYYY-MM-DD"
{
  date: "2026-03-27",
  raw: "【14:30】今天开了个会\n\n---\n\n【18:45】晚上吃了火锅",
  formatted: "## 今日工作\n- (14:30) 开会...",
  updatedAt: 1743040800000
}
```

---

## AI Prompt 规则

- 输入：`rawAccumulated`（含时间戳的全量原始记录）
- 时间处理：`【HH:MM】` 为提交时间；用户文本中明确说明的时间优先，转24小时制
- 输出格式：`## 今日工作 / ## 想法·灵感 / ## 知识碎片 / ## 复盘`，无内容的章节省略
- 每个条目前加 `(HH:MM)` 标记
