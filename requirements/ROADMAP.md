# 产品路线图

**产品愿景**：让记日记变成一件顺手的事——用户只需随口说几句，AI 自动整理成结构化日记。

---

## Phase 1 — MVP 验证期（当前阶段）

目标：以最小成本验证"AI 降低记录门槛"这个核心假设是否成立。

### 已完成 ✅

- [x] 今日记录页（多次追加输入，AI 流式整理，自动保存）
- [x] 每条记录自动打时间戳，AI 识别用户自述时间
- [x] 整理时内容变暗、流式替换动画（不闪烁）
- [x] Markdown 渲染（标题、列表、分割线、加粗等）
- [x] 手动编辑已整理内容
- [x] 用原始记录重新整理（重跑 AI）
- [x] 原始记录可折叠展示
- [x] 历史日记列表（折叠/展开，Markdown 渲染）
- [x] API Key 设置与本地保存
- [x] 清除所有数据
- [x] GitHub Pages 自动部署
- [x] 移动端适配（底部输入栏、安全区、字体）

## Phase 2 — 体验优化期

目标：打磨核心体验，提升留存。

- [ ] **后台日记 Agent**（高优先级）：独立后端服务，提供专业 Agent 替代当前"全量重写"模式。Agent 每次只处理新增内容并增量更新日记，避免重复整理已有内容，提升速度与稳定性。详见 [RFC-008](./changes/RFC-008-diary-agent-backend.md)
- [ ] 目标管理：用户设定每日/周目标，AI 在复盘时对比行为与目标的偏差
- [ ] 语音输入：调用浏览器原生语音 API，转文字后触发整理
- [ ] 日记模板：用户可自定义 AI 整理的输出格式
- [ ] 搜索：在历史日记中按关键词搜索
- [ ] 数据导出：将所有日记导出为 Markdown 文件

---

## Phase 3 — 规模化

目标：从个人工具演进为多用户产品，上架华为应用市场。

- [ ] **华为应用市场上架**：HarmonyOS 原生 App 或 Web 套壳打包，目标平台：鸿蒙 + Android
- [ ] 用户账号与云同步（数据不再依赖本地 localStorage）
- [ ] 多设备同步
- [ ] 分享：生成单篇日记的公开链接

---

## 暂缓 / 放弃

| 功能 | 状态 | 原因 |
|------|------|------|
| 原生 App（MVP 阶段） | 暂缓 | 先用 H5 验证需求，成本低 |
| 服务端代理 API Key | 暂缓 | MVP 个人工具，用户自持 Key 可接受 |
| 目标管理（MVP 阶段） | 暂缓 | 功能复杂，先聚焦记录本身 |
| 语音录入（MVP 阶段） | 暂缓 | 输入法自带语音转文字可代替 |
| 社交 / 分享（MVP 阶段） | 暂缓 | 日记属于私密内容，不急于社交化 |
